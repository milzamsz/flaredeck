# Architecture Decision Prompt

Prepare an ADR for a material FlareDeck decision. Do not implement the decision in the same task unless explicitly approved after review.

## Decision input

- Problem and business/developer impact
- Current constraints
- Options considered
- Security and compatibility implications
- Migration and rollback implications
- Trigger or urgency

## Required analysis

Evaluate each option for:

- alignment with product scope;
- complexity and operational burden;
- security and trust boundaries;
- cross-platform support;
- maintainability and testability;
- release and migration impact;
- dependency and lock-in;
- simpler alternative;
- future evolution path.

## ADR output

- Title
- Status: Proposed
- Context
- Decision drivers
- Options
- Decision
- Positive consequences
- Negative consequences
- Rejected alternatives
- Security implications
- Migration/rollback
- Validation plan
- Change triggers

Do not disguise a preferred implementation as the only possible option.
